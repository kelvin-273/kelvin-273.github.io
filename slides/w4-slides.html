The general function pattern.

# def function(x,y):
function: 
    # save $ra and $fp and update $fp
    addi $sp, $sp, -8 # make space for $fp and $ra
    sw $fp, 0($sp)    # store $fp on stack
    sw $ra, 4($sp)    # store $ra on stack
    addi $fp, $sp, 0  # copy $sp to $fp

    # use $v0 to store result of computation
    
    # return
    lw $fp, 0($sp)    # restore $fp
    lw $ra, 4($sp)    # restore $ra
    addi $sp, $sp, 8  # remove space on stack for $ fp and $ra
    jr $ra            # return to address pointed to by $ra

Core logic without returns

# def function(x,y):
function: 
    # save $ra and $fp and update $fp

    # use $v0 to store result of computation
    # if x >= y
    lw $t0, 8($fp)      # $t0 = x
    lw $t1, 12($fp)     # $t1 = y

    bne $t2, $0, endif  # If not x>=y go to end if

Q3

Rational behind the the function call convention

  1. Caller: saves temporary registers by pushing their values on to the stack
  2. Caller: prepares the arguments by pushing them on to the stack
  3. Caller: calls the function using the jal instruction
  4. Callee: saves $ra by pushing its value on the stack
  5. Callee: saves $fp by pushing its value on the stack
  1. Callee: copies $sp to $fp
  2. Callee: allocates local variables by reserving enough space onto the stack
  3. Callee: if there is a return value, stores it in $v0
  4. Callee: deallocates local variables by popping the previously pushed space
  5. Callee: restores $fp by popping its saved value off the stack
  1. Callee: restores $ra by popping its saved value off the stack
  2. Callee: returns using the jr $ra instruction
  3. Caller: clears the function arguments by popping their allocated space off the stack
  4. Caller: restores temporary registers by popping their values off the stack
  5. Caller: uses the return value $v0 if necessary
